Brief Técnico: Proyecto "OmniReport AI" (SaaS Multi-tenant)Objetivo: Construir un motor de generación de reportes técnicos automatizados que procese audio y fotos mediante IA Multimodal.1. Arquitectura y EstándaresPatrón: Clean Architecture (Independencia de frameworks, UI y DB).Capas:Domain: Entidades (Report, User, Client) y Reglas de Negocio.Use Cases: Logic para GenerateReport, ProcessMedia, ManageSubscription.Infrastructure: Adaptadores para PostgreSQL, Gemini API, ElevenLabs, y Railway.Frontend: Progressive Web App (PWA) enfocada en "Mobile-First" para uso en terreno.2. Stack TecnológicoBackend: Node.js (TypeScript) o Go (a elección del agente para máximo rendimiento).Base de Datos: PostgreSQL con esquema de aislamiento de datos (Multi-tenancy via organization_id).IA Multimodal: Gemini 1.5 Pro (para análisis de imagen y razonamiento de texto).Procesamiento de Voz: Whisper o Gemini (STT) para transcripción de audio técnico.Infraestructura: Dockerizado para despliegue inmediato en Railway.app.3. El "Pipeline" de Datos (Core Logic)El agente debe implementar el siguiente flujo asíncrono:Ingesta: Captura de $N$ imágenes y 1 archivo de audio (m4a/mp3).Transcripción: Convertir audio a texto técnico, limpiando muletillas.Análisis Multimodal: Enviar el texto transcrito + las imágenes a Gemini con un System Prompt que actúe como experto en el nicho (ej. Mecánica o Construcción).Generación de Output: Retornar un JSON estructurado que contenga: Hallazgos, Gravedad, Acción Recomendada y Resumen Ejecutivo.Exportación: Generación dinámica de PDF profesional y vista web compartible.4. Estructura de Datos Inicial (Entidades)User: Auth y perfiles.Organization: El "Tenant" (cada taller o empresa).Report: ID, metadata, transripciones, URLs de imágenes en S3/R2, y el resultado final de la IA.Template: Configuración de prompts específicos por industria.
